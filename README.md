# Container 期中專題 — 分散式共享帳本

## 專題目標

透過 Docker 容器技術，實作一個簡易的**分散式帳本系統**，具備基本區塊鏈概念，包含交易紀錄、區塊串接與完整性驗證，並模擬多個 Client 同步操作帳本的情境。

---

## 系統功能需求

| 功能 | 程式 | 說明 |
|------|------|------|
| 查詢帳戶餘額 | `app_checkMoney.py <帳戶>` | 查詢指定帳戶目前的餘額 |
| 查詢交易紀錄 | `app_checkLog.py <帳戶>` | 列出指定帳戶的所有交易歷史 |
| 進行轉帳 | `app_transaction.py <來源> <目標> <金額>` | 帳戶間金額轉移，記錄於區塊中 |
| 驗證帳本完整性 | `app_checkChain.py` | 檢查所有區塊的 SHA256 串接是否正確 |

---

## 系統設計規範

- 可使用任意程式語言開發
- 所有帳本資料儲存在共用資料夾（`/share` 或 `/storage`）
- 每 **5 筆交易**組成一個區塊（Block），以文字檔儲存（`1.txt`, `2.txt`, ...）
- 每個區塊包含：
  - 前一區塊的 SHA256 雜湊值
  - 下一區塊的連結資訊
  - 該區塊的所有交易紀錄（格式：`來源, 目標, 金額`）
- 區塊達到 5 筆交易上限時，系統**自動建立新區塊**並計算前一區塊 SHA256
- `app_checkChain.py` 驗證完整後，執行者可從 `angel` 得到 **10 元獎勵**

### 區塊格式範例

```
Sha256 of previous block: <hash>
Next block: 2.txt
angel, A, 100
angel, B, 100
angel, C, 100
A, B, 10
A, C, 20
```

---

## 系統架構

```
Client 1          Client 2          Client 3
   |                 |                 |
/storage          /storage          /storage
(共用帳本區塊 1~N，三個節點同步)
```

3 個 Docker Container 共享同一份帳本資料夾。

## Demo 前準備
