真分散式架構 加分

物理隔離儲存空間

  • 每個節點獨立 Named Volume（非 bind mount）
  • 三個 volume 完全物理隔離，資料不共享
  • 即使同一台主機，帳本也真正分散

HTTP 即時同步機制

  • 寫入後 POST /sync/block 廣播給所有 peer
  • 衝突策略：先到先得（HTTP 409 拒絕）
  • fcntl.flock 並發保護，POSIX 標準鎖

三節點各持有完整帳本副本，任一節點當機不影響整體服務

跨節點完整性驗證與自動修復 加分

多數決演算法

  • 蒐集所有節點的每個區塊 computed hash
  • 同一位置 hash 相符 ≥ 2/3 → 視為正確版本
  • 少數派節點用正確版本覆蓋並修復鏈
  • 可偵測「內容被竄改但 hash 欄位未更新」情境

驗證流程

  • Step 1：對自身鏈做 SHA-256 逐塊驗證
  • Step 2：呼叫 GET /chain/hashes 蒐集各節點
  • Step 3：多數決找出正確版本
  • Step 4：自動修復 + 稽核日誌記錄
一鍵完成：單節點損壞 → 自動修復 → 重新驗證通過

F1：動態節點加入 加分

設計理念

  • 節點運營者自行啟動 node4 容器，
    無需修改 docker-compose.yml
  • 向任一現有節點 POST /nodes/join 申請加入
  • 管理員在 Web UI 審核，核准後廣播全網
  • 新節點自動拉取完整鏈並開始參與同步

流程步驟

  • ① node4 容器啟動 → POST /nodes/join
  • ② 加入 PENDING_NODES（持久化至 volume）
  • ③ Admin 在 Nodes 頁面點擊「核准」
  • ④ 廣播 /nodes/welcome → node4 接收完整鏈
  • ⑤ node4 加入 KNOWN_PEERS，正常同步
